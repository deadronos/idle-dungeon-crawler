# 001 - Player Classes and Party System

**Date:** 2026-03-10
**Status:** Accepted

## Context
The game started as a simple clicker game where the player clicked an enemy to deal damage. To create a deeper and more engaging RPG experience, we needed a system that allowed for varied gameplay styles, strategic team composition, and idle progression. We needed to transition from an active clicker to an autobattler.

## Decision
We implemented a Party System where the player manages a roster of heroes.
1.  **Classes:** We introduced distinct hero classes (Warrior, Cleric, Archer), each with unique attributes, derived stats, and secondary resources.
    *   **Warrior:** Uses Rage (generated by attacking), focuses on Strength and Vitality.
    *   **Cleric:** Uses Mana (regenerates slowly), focuses on Intelligence and Wisdom, primarily heals.
    *   **Archer:** Uses Cunning (rapid regeneration), focuses on Dexterity, deals high critical damage.
2.  **Attributes:** We defined a core set of attributes: Constitution (VIT), Strength (STR), Dexterity (DEX), Intelligence (INT), and Wisdom (WIS).
3.  **Active Time Battle (ATB):** We replaced manual clicking with an ATB system. Every entity (player and enemy) has an action progress bar that fills over time, triggering attacks or skills when full.
4.  **Auto-Progress:** We added a toggle to automatically progress to the next dungeon floor upon defeating all enemies in a room.
5.  **Party Wipe:** If the entire party dies, they are fully healed but reset to Floor 1, losing all current gold (a rogue-lite prestige mechanic foundation).

## Consequences
*   **Easier:** Designing new classes or enemies is straightforward by adjusting base attributes and resource types. The game can now be played entirely idly.
*   **Difficult:** Balancing becomes much more complex, requiring careful tuning of attribute multipliers, exponential EXP curves, and enemy scaling to ensure the game doesn't become too easy or impossible at higher floors. It also requires more complex UI to display party state.
